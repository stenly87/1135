# Лабораторная работа: Паттерн проектирования "Состояние" (State) 
Описание паттерна https://metanit.com/sharp/patterns/3.6.php
## Тема: Логика управления автономным складским роботом (AGV)

### 1. Описание проблемы
Вы разрабатываете ПО для управления автономным складским роботом. Робот имеет сложный жизненный цикл: он может ожидать задачу, двигаться к точке, загружать груз, разгружать его, заряжаться или находиться в режиме ошибки.

Поведение робота сильно зависит от его текущего состояния. Например:
*   Команда `Move()` валидна только когда робот не заряжается и не загружен грузом сверх нормы.
*   Команда `Pickup()` (взять груз) невозможна, если робот движется или находится на зарядке.
*   При низком уровне заряда робот должен прервать текущую задачу и перейти в состояние зарядки, игнорируя новые задачи.

**Задача:** Реализовать управление роботом, избегая огромных конструкций `if-else` или `switch-case` внутри методов контекста. Использование паттерна **State** должно позволить легко добавлять новые состояния (например, "Режим диагностики") без изменения существующего кода контекста.

### 2. Требования к приложению
*   **Язык:** C# (.NET 6/8).
*   **Тип приложения:** Консольное.
*   **Интерфейс:** Пользователь вводит команды (например, "start task", "low battery", "error"), программа выводит реакцию робота и текущее состояние.
*   **Архитектура:** Строгое разделение на Контекст и Состояния.

### 3. Схема классов (UML-описание)

Вам необходимо спроектировать следующую структуру:

#### 3.1. Интерфейс состояния (`IRobotState`)
Определяет действия, которые могут быть вызваны у робота. Каждое конкретное состояние решает, как реагировать на эти действия.
*   `Handle(Context robot)`: (Опционально) метод для инициализации при входе в состояние.
*   `Move()`: Попытка перемещения.
*   `PickupCargo()`: Попытка захвата груза.
*   `DropCargo()`: Попытка выгрузки груза.
*   `Charge()`: Попытка подключения к зарядке.
*   `FinishTask()`: Завершение текущей операции.
*   `HandleError()`: Симуляция поломки.

#### 3.2. Контекст (`WarehouseRobot`)
Хранит текущее состояние и данные робота (уровень заряда, наличие груза, координаты).
*   **Поля:**
    *   `_currentState : IRobotState`
    *   `BatteryLevel : int`
    *   `HasCargo : bool`
*   **Методы:**
    *   `SetState(IRobotState newState)`: Меняет состояние, может вызывать метод входа в состояние.
    *   Публичные методы-обертки (`Move()`, `PickupCargo()` и т.д.), которые делегируют вызов текущему состоянию (`_currentState.Move()`).
    *   Геттеры для данных состояния (чтобы состояния могли проверять условия).

#### 3.3. Конкретные состояния (Concrete States)
Каждый класс реализует `IRobotState`. Они содержат логику переходов.
1.  **`IdleState`** (Ожидание): Робот стоит на базе. Может принять задачу (переход в `MovingToTarget`), может уйти на зарядку (если батарея низкая).
2.  **`MovingToTargetState`** (Движение к цели): Имитация движения. Нельзя брать груз. При низком заряде переход в `ChargingState`. При прибытии переход в `LoadingState`.
3.  **`LoadingState`** (Загрузка): Имитация подъема груза. Нельзя двигаться. После успеха переход в `MovingToDeliveryState`.
4.  **`MovingToDeliveryState`** (Движение к точке выгрузки): Аналогично движению к цели, но с грузом.
5.  **`UnloadingState`** (Выгрузка): Сброс груза. После успеха переход в `IdleState`.
6.  **`ChargingState`** (Зарядка): Игнорирует задачи движения. Ждет полного заряда, затем переходит в `IdleState`.
7.  **`ErrorState`** (Ошибка): Блокирует все действия кроме перезагрузки (`Reset`).

### 4. Порядок выполнения

1.  **Реализация каркаса:**
    *   Создайте интерфейс `IRobotState` и класс `WarehouseRobot`.
    *   Реализуйте 4 состояния (например: `Idle`, `Moving`, `Charging`, `Error`).
2.  **Логика переходов:**
    *   Реализуйте автоматический переход в `ErrorState`, если уровень заряда падает ниже 0% во время движения.
    *   Реализуйте блокировку действий (например, вывод сообщения "Нельзя двигаться во время зарядки").
3.  **Демонстрация:**
    *   Напишите в `Program.cs` сценарий, который последовательно переводит робота через несколько состояний, выводя логи в консоль.
    *   Пример лога:
        ```text
        [Robot] State: Idle
        [User] Command: Start Task
        [Robot] State: MovingToTarget... Arrived.
        [Robot] State: Loading...
        [User] Command: Low Battery Signal
        [Robot] State: Charging (Task Interrupted!)
        ```
