# Лабораторная работа: Паттерн «Декоратор» (Decorator)
## Тема: «Арсенал Магического Кузнеца»

### 1. Легенда
Вы разрабатываете подсистему инвентаря для новой RPG-игры. В игре существует множество видов оружия, которое игроки могут улучшать у кузнеца. Улучшения могут быть физическими (заточка, балансировка) и магическими (руны, зачарования).

**Проблема:** Количество комбинаций улучшений огромно. Если использовать наследование для каждого варианта (например, `FireSharpSword`, `VampiricSharpFireSword`), классы размножатся экспоненциально, и код станет неподдерживаемым.

**Задача:** Реализовать систему расчета характеристик оружия, позволяющую динамически «навешивать» улучшения на любой базовый предмет во время выполнения программы, не создавая новых классов для каждой комбинации.

### 2. Требования к функционалу
Консольное приложение на C# должно демонстрировать работу паттерна.

1.  **Базовые характеристики:** У каждого предмета есть:
    *   Название (Description).
    *   Сила урона (Damage).
    *   Стоимость (Price).
2.  **Динамическое улучшение:** Игрок может применить к оружию несколько улучшений подряд. Каждое улучшение меняет характеристики (увеличивает урон, меняет цену, добавляет текст к описанию).
3.  **Вывод информации:** В консоли должно быть четко видно, как меняется описание предмета по мере наложения декораторов (цепочка оберток).
4.  **Запрет на комбинаторное наследование:** Нельзя создавать классы типа `EnchantedSharpenedSword`. Все комбинации должны собираться через декораторы.

### 3. Схема классов (Class Diagram Description)

Вам необходимо спроектировать и реализовать следующую структуру:

#### 3.1. Компонент (Component)
**Интерфейс `IWeapon`**
Определяет контракт для всех объектов, которые можно улучшать.
*   Метод `string GetDescription()`: Возвращает текстовое описание предмета.
*   Метод `int GetDamage()`: Возвращает итоговый урон.
*   Метод `decimal GetPrice()`: Возвращает итоговую стоимость.

#### 3.2. Конкретный компонент (Concrete Component)
**Класс `BaseSword` (реализует `IWeapon`)**
Базовое оружие без улучшений.
*   Поля: `_baseDamage`, `_basePrice`, `_name`.
*   Конструктор: Принимает базовые значения.
*   Реализация методов: Возвращает базовые значения без модификаций.
*   *Пример:* «Ржавый меч», Урон: 10, Цена: 50.

*Добавить `BaseAxe` или `BaseStaff` для демонстрации полиморфизма.*

#### 3.3. Декоратор (Decorator)
**Абстрактный класс `WeaponDecorator` (реализует `IWeapon`)**
Хранит ссылку на объект типа `IWeapon`.
*   Поле: `protected IWeapon _decoratedWeapon`.
*   Конструктор: Принимает объект `IWeapon` и сохраняет его.
*   Реализация методов `IWeapon`: По умолчанию делегирует вызовы внутреннему объекту `_decoratedWeapon` (чтобы поведение не менялось, если декоратор пустой).
    *   `GetDescription()` -> возвращает `_decoratedWeapon.GetDescription()`.
    *   `GetDamage()` -> возвращает `_decoratedWeapon.GetDamage()`.
    *   `GetPrice()` -> возвращает `_decoratedWeapon.GetPrice()`.

#### 3.4. Конкретные декораторы (Concrete Decorators)
Необходимо реализовать минимум **3 разных декоратора**, каждый из которых меняет логику методов:

1.  **`SharpeningDecorator` (Заточка)**
    *   Логика: Увеличивает физический урон на фиксированное значение (например, +5).
    *   Цена: Увеличивается на 20%.
    *   Описание: Добавляет суффикс " [Заточено]".

2.  **`FireRuneDecorator` (Руна Огня)**
    *   Логика: Добавляет магический урон (например, +10 к общему урону).
    *   Цена: Увеличивается значительно (например, +100 монет).
    *   Описание: Добавляет суффикс " [Огненная руна]".

3.  **`VampirismEnchantDecorator` (Зачарование Вампиризма)**
    *   Логика: Немного снижает базовый урон (например, -2), но добавляет эффект (для упрощения вывод в консоль).
    *   Цена: Увеличивается в 2 раза (редкое зачарование).
    *   Описание: Добавляет суффикс " [Вампиризм]".

### 4. Сценарий работы программы (Main)
В методе `Main` необходимо продемонстрировать следующие сценарии:

1.  Создание базового меча. Вывод его характеристик.
2.  Применение заточки. Вывод характеристик (видно изменение цены и описания).
3.  Применение Руны Огня к *уже заточенному* мечу. Вывод характеристик.
4.  Создание нового меча и применение на него Вампиризма, а затем Заточки (другой порядок). Вывод характеристик.
5.  **Важно:** В описании (`GetDescription`) должна быть видна цепочка.
    *   *Пример вывода:* `Ржавый меч [Заточено] [Огненная руна]`
    *   *Пример вывода:* `Ржавый меч [Вампиризм] [Заточено]`

### 5. Пример ожидаемого вывода в консоль
```text
--- Базовое оружие ---
Предмет: Стальной меч
Урон: 20
Цена: 100.00

--- После заточки ---
Предмет: Стальной меч [Заточено]
Урон: 25
Цена: 120.00

--- После заточки и Руны Огня ---
Предмет: Стальной меч [Заточено] [Огненная руна]
Урон: 35
Цена: 220.00

--- Другой порядок (Вампиризм -> Заточка) ---
Предмет: Стальной меч [Вампиризм] [Заточено]
Урон: 23
Цена: 240.00
```
